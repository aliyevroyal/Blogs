O.O.P. and S.O.L.I.D. principles

 In 1967, O.O.P. (Object-oriented Programming) paradigm was invented by Alan Kay. Almost everything about programming changed essentially and nothing would be the equivalent from there on. Since it was coined till today O.O.P. has been helping us to relate programming to objects of real world as a idea. So, it gives us better opportunity to plan and design architecture of our codes cleaner, easier and more flexible.

 Nowadays popular and modern programming languages (as Java, C# etc.) lets us to write our codes due to O.O.P. paradigm. But most of the novices have misunderstandings. Every code is not O.O.P. just because it is written in programming language that based on or support O.O.P. paradigm.

 In concisely, we can say that code is “bad” if;
• Any update request came from client is hard to carry out
• Changing even small part of code make everything messy
• The same code piece/block used in several different parts of project

 If any of these signs appeared in our code we ought to redesign our code. Luckily, we do not have to reinvent wheel to write “good” code.
